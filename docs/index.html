x.ui is based on the scss framework xlcss. It is inspired by the unicorn workflow.
the concept is based on a modular approach.

basics

files

x.ui and xlcss are the solid foundation of a project based on an unicorn workflow. x.ui sets the basic elements, which can be combined top bigger elements like partials or templates. In _config.scss, you can adjust several variables for colors, sizes or font-face. Those variables then are used throuout the project.

The first four layers should be highly adaptable to other projects. Those Layers are core-elements, global-elements, marco-elements and presets


  x.ui/
  ├── _config.scss
  ├── core/
  ├── presets/
  ├── global/
  ├── macros/
  ...

the layers partials and templates are reserved for the specific project.

  ├── partials/
  └── templates/

core

core styles, are styles that don't refer a specific DOM object, but the whole project in general like colors and basic font settings
