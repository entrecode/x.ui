get started

welcome to
the atomsphere.

x.ui is a framework based on the atomic design methodology. this workflow is inspired by nature and physics and has a pretty serious focus on modularity and structure. it structure begins with the smallest part and builds up to complex structure. as a result, there is very little redundancy with a remarkable flexibility.

the goal of x.ui is to translate the atomic philosophy into a new kind of framework, unattached to design-trends and 100% focused on user-driven ux-design and combine it a with a strict set of rules regarding naming conventions and front-end development.

BEM

{block-name}__{elem-name}_{mod-name}_{mod-val}

a good structure also needs as intuitive set of rules for class names to use. For this x.ui follows the BEM methodology (Block Element Modifier). The rules are:

  • names are written in lowercase Latin letters.
  • words are separated by a hyphen (-).
  • the element name is separated from the block name by a double underscore (__).
  • the modifier name is separated from the block or element name by a single underscore (_).
  • the modifier value is separated from the modifier name by a single underscore (_).

Source: Naming convention / Methodology / BEM - https://en.bem.info/methodology/naming-convention/ - 13.12.2017

structure

atoms

Atoms are the smallest building blocks of x.ui. Atoms are elements like type, headings, inputs, buttons and more. Atoms only have one function. Most atoms have specific modifiers that change attributes like size, style or color.

molecules

Molecules are groups of atoms bonded together and are the smallest fundamental units of a compound. molecules group connected information and/or interactions.

organisms

organisms combine complex groups of molecules and molecules. they are the biggest parts of the atomsphere.

x.ui is mobile first. So if you don't combine multiple breakpoints, the xs prefix is not needed. breakpoint shortcodes can post-fixed to most of classes or attribute-values. normally it means that the style takes effect from the breakpoint up. A few styles just use a specific breakpoint area (e.g. from xs to xs-max).

the breakpoints are defined in the $breakpoints map. So they can easily be overwritten with custom breakpoints and shortcodes.


$breakpoints: (
  'xs': 0,
  'sm': 480,
  'md': 720,
  'lg': 1024,
  'xl': 1280,
  'xxl': 1920,
) !default;

gaps & spacing

gaps and spacing are one of the most important elements of an user interface. Following the gestalt psychology, more precisely the law of proximity, they give structure and illustrate the relation of functions and informations to each other

x.ui uses a simple system for all structural atoms and modifier. There are 9 different spacing units

postfix description
-0 no size, used to reset spacing for a specific media breakpoint.
-1 [4px] - small gutter for atoms that belong together or elements within an atom.
-2 [8px] - normal gutter for atoms that belong together or elements within an atom.
-3 [16px] - small gutter for atoms within a molecule or organisms
-4 [24px] - normal gutter for atoms within a molecule or organisms
-5 [32px] - small gutter for molecule or organisms
-6 [48px] - normal gutter for molecule or organisms
-7 [64px] - small gutter for organisms, that have no relation
-8 [80px] - very big gutter for organisms, that have no relation

Additional flex attributes

Those additional attributes and values can be combined with data-grid / data-col to modify the layout.

data-flex="{option}{ -breakpoint | null }"
flex-direction
is-row left to right in ltr | right to left in rtl default
is-row-reverse right to left in ltr | left to right in rtl
is-column same as row but top to bottom
is-column-reverse same as row-reverse but bottom to top
flex-wrap
is-nowrap all flex items will be on one line default
is-wrap flex items will wrap onto multiple lines, from top to bottom
is-wrap-reverse flex items will wrap onto multiple lines from bottom to top
justify-content
is-start items are packed toward the start line default
is-end items are packed toward to end line
is-center items are centered along the line
is-between items are evenly distributed in the line; first item is on the start line, last item on the end line
is-around items are evenly distributed in the line with equal space around them
align-items
is-start-items cross-start margin edge of the items is placed on the cross-start line
is-end-items cross-end margin edge of the items is placed on the cross-end line
is-center-items items are centered in the cross-axis
is-baseline-items items are aligned such as their baselines align
is-stretch-items stretch to fill the container default
align-content
is-start-content lines packed to the start of the container
is-end-content lines packed to the end of the container
is-center-content lines packed to the center of the container
is-space-content lines evenly distributed; the first line is at the start of the container while the last one is at the end
is-arround-content lines evenly distributed with equal space around each line
is-stretch-content lines stretch to take up the remaining space default
data-flex-item="{options}{ -breakpoint | null }"
size
grow items grows to fill it's parent.
grow-even all children grow evenly an share the space.
fit-self element doesn't shrink and grows up to its content size.
align-self
is-auto Default. The element inherits its parent container's align-items property, or "stretch" if it has no parent container.
is-stretch The element is positioned to fit the container.
is-center The element is positioned at the center of the container.
is-start The element is positioned at the beginning of the container.
is-end The element is positioned at the end of the container.
is-baseline The element is positioned at the baseline of the container.
data-flex-order="{order}{ -breakpoint | null }"

By default, flex items are laid out in the source order. However, the order property controls the order in which they appear in the flex container. the order-value can be [1-12].

the has-height modifier is the vertical layout equivalent for columns. size modifier set the height of an element relative to its parent or to the viewport.

{relation}
  • empty - relative to it's parent (%)

  • half - × .5
  • 3-4 - × .75
  • golden - × .5625
