welcome to the atomsphere.

x.ui is a framework based on the atomic design methodology. this workflow is inspired by nature and physics and has a pretty serious focus on modularity and structure. it structure begins with the smallest part and builds up to complex structure. as a result, there is very little redundancy with a remarkable flexibility.

the goal of x.ui is to translate the atomic philosophy into a new kind of framework, unattached to design-trends and 100% focused on user-driven ux-design and combine it a with a strict set of rules regarding naming conventions and front-end development.

BEM

block-name__elem-name_mod-name_mod-val

a good structure also needs as intuitive set of rules for class names to use. For this x.ui follows the BEM methodology (Block Element Modifier). The rules are:

  • names are written in lowercase Latin letters.
  • words are separated by a hyphen (-).
  • the element name is separated from the block name by a double underscore (__).
  • the modifier name is separated from the block or element name by a single underscore (_).
  • the modifier value is separated from the modifier name by a single underscore (_).

Source: Naming convention / Methodology / BEM - https://en.bem.info/methodology/naming-convention/ - 13.12.2017

structure

atoms

Atoms are the smallest building blocks of x.ui. Atoms are elements like type, headings, inputs, buttons and more. Atoms only have one function. Most atoms have specific modifiers that change attributes like size, style or color.

molecules

Molecules are groups of atoms bonded together and are the smallest fundamental units of a compound. molecules group connected information and/or interactions.

organisms

organisms combine complex groups of molecules and molecules. they are the biggest parts of the atomsphere.

data-container="{option}"

wraps everything in a repsonsive container with a maximum with of 1200px. Per default it is centered horizontally. for full-width designs just don't use this attribute.

empty centered horizontally
default
left aligns the container left
right aligns the container right
data-grid="{option}-{breakpoint}"

this is part of the flex-based grid. it is a more modern and flexible approach for a grid-model which is not prone to floating errors. it can be combined with additional data-flex values to influence it's behavior.

empty activates flex-gird
default
no-gutter removes the space between the inline columns
small half sized spaces between columns
center centers the columns
data-col="{size | offset}-{breakpoint}"
[1-12] defines the grid-size of the column. can be multiple sizes for different screen-sizes
+[0-11] defines an grid-based offset of an column. can be multiple offsets for different screen-sizes
data-flex="{option}-{breakpoint}"
flex-direction
row left to right in ltr | right to left in rtl default
row-reverse right to left in ltr | left to right in rtl
column same as row but top to bottom
column-reverse same as row-reverse but bottom to top
flex-wrap
nowrap all flex items will be on one line default
wrap flex items will wrap onto multiple lines, from top to bottom
wrap-reverse flex items will wrap onto multiple lines from bottom to top
justify-content
start items are packed toward the start line default
end items are packed toward to end line
center items are centered along the line
between items are evenly distributed in the line; first item is on the start line, last item on the end line
around items are evenly distributed in the line with equal space around them
align-items
start-items cross-start margin edge of the items is placed on the cross-start line
end-items cross-end margin edge of the items is placed on the cross-end line
center-items items are centered in the cross-axis
baseline-items items are aligned such as their baselines align
stretch-items stretch to fill the container default
align-content
start-content lines packed to the start of the container
end-content lines packed to the end of the container
center-content lines packed to the center of the container
space-content lines evenly distributed; the first line is at the start of the container while the last one is at the end
arround-content lines evenly distributed with equal space around each line
stretch-content lines stretch to take up the remaining space default
data-flex-item="{options}-{breakpoint}"
size
grow items grows to fill it's parent.
grow-even all children grow evenly an share the space.
fit-self element doesn't shrink and grows up to its content size.
data-flex-order="{order}-{breakpoint}"

By default, flex items are laid out in the source order. However, the order property controls the order in which they appear in the flex container. the order-value can be [1-12].

data-row
deprecated

is an element of the float grid system, which is the old standard for grid-systems. to avoid floating-errors it should only contain grid-columns with a maximum width of 12 columns. Per default data-row doesn't need a value. Optional values are no-gutter to remove the space between the inline columns or small-grid for half sized spaces between columns.

spacing

{state}-{property}-{sides}-{size}
{state} has or no
{property} either padding or margin
{sides}
  • no value - adds margin or padding to all sides
  • middle - top and bottom
  • center - left and right
  • top right bottom left
{size}
  • no value - default size $base-spacing
  • small - $base-spacing / 2
  • big - $base-spacing * 2
  • auto - only with margin-l margin-r margin-center

has-height-{relation}-{ratio}-{breakpoint}

size modifier are the vertical layout equivalent for columns. size modifier set the min-height of an element relative to its parent or to the viewport.

{relation}
  • empty - relative to it's parent (%)
  • vw - relative to viewport width
  • vh - relative to viewport height
{ratio}
  • full - × 1
  • 1-4 - × .25
  • half - × .5
  • 3-4 - × .75
  • golden - × .5625
  • 16-9 - × .6180 (golden-ratio)
