basics

files

x.ui and xlcss are the solid foundation for any project based on an unicorn workflow. x.ui sets the basic elements, which can be combined to bigger elements like partials or templates. In _config.scss, you can adjust several variables for colors, sizes or font-face. Those variables then are used throuout the project.

The first four layers should be highly adaptable to other projects. Those Layers are core-elements, global-elements, marco-elements and presets


  x.ui/
  ├── _config.scss
  ├── _validator.scss
  ├── _core/
  ├── _presets/
  ├── _global/
  ├── _macros/
  ...

the layers partials and templates are reserved for the specific project.


  ...
  └── project/
      ├── macros/
      ├── partials/
      └── templates/

naming conventions

x.ui follows the BEM (Block, Element, Modifier) approach. For more see https://en.bem.info/methodology/quick-start/.

Modifier

An entity that defines the appearance, state, or behavior of a block or element. With x.ui there are some global modifier classes.
